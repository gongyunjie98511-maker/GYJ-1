# 第二章：Cursor —— AI 加持的代码编辑器

> 本章从零开始介绍 Cursor 的安装、界面、AI 核心功能与实用技巧，帮助你把 AI 真正融入编程日常。

---

## 2.1 Cursor 是什么

### 2.1.1 Cursor 的定位

Cursor 是一款**内置 AI 的代码编辑器**，由 Anysphere 公司开发。它的底层基于 VS Code（微软的开源编辑器），在此基础上深度集成了 AI 能力，让写代码变得像聊天一样自然。

**一句话理解：** Cursor = VS Code + 超强 AI 副驾驶

### 2.1.2 Cursor 能做什么

| 功能 | 描述 |
|------|------|
| Tab 智能补全 | 预测你下一步要写什么，Tab 键一键接受 |
| AI 对话（Chat）| 在编辑器侧边跟 AI 聊代码，可直接引用文件 |
| AI 编辑（Composer）| 告诉 AI 你要做什么，它直接帮你修改代码 |
| 全代码库问答 | 问"这个函数在哪里定义的"，AI 帮你找 |
| 错误修复 | 选中报错代码，一键让 AI 解释并修复 |

### 2.1.3 Cursor 与 VS Code 的关系

- Cursor **兼容 VS Code** 的所有插件、主题、配置
- 从 VS Code 迁移到 Cursor 几乎零成本，设置一键导入
- Cursor 在 VS Code 基础上新增了 AI 面板和快捷键

## 2.2 下载与安装

### 2.2.1 下载

访问官网下载：`https://www.cursor.com`

选择对应系统版本（Windows / Mac / Linux），下载安装包。

### 2.2.2 Windows 安装步骤

1. 双击下载好的安装包（如 `Cursor Setup x.x.x.exe`）
2. 选择安装路径（建议安装到非 C 盘，例如 `E:\软件\cursor`）
3. 等待安装完成，桌面出现 Cursor 图标

### 2.2.3 首次启动配置

首次打开 Cursor 会引导你：

1. **导入 VS Code 配置**（如已安装 VS Code，可一键导入插件和设置）
2. **选择 AI 模型**（默认 claude-3.5-sonnet，推荐保持默认）
3. **登录账号**（注册 Cursor 账号，新用户有免费 Trial 额度）

## 2.3 界面介绍

### 2.3.1 两种界面模式

Cursor 有两种主界面模式，初学者容易混淆：

| 模式 | 描述 | 菜单语言 |

| **经典编辑器**（Editor）| 传统代码编辑界面，类似 VS Code | 中文（安装语言包后）|

## 2.4 中文配置

### 2.4.1 安装中文语言包

在 Cursor 内置终端中执行：

```powershell
cursor --install-extension MS-CEINTL.vscode-language-pack-zh-hans
```

安装完成后重启 Cursor，经典编辑器菜单即变为中文（文件 / 编辑 / 查看 …）。

### 2.4.2 强制设置中文语言（可选）

如果重启后仍是英文，手动指定语言：

**用户语言文件：**
路径：`C:\Users\你的用户名\AppData\Roaming\Cursor\User\locale.json`

```json
{
  "locale": "zh-cn"
}
```

**应用级语言文件：**
路径：`C:\Users\你的用户名\.cursor\argv.json`

```json
{
  "locale": "zh-cn"
}
```

### 2.4.3 为什么部分界面仍显示英文

- **Glass / Agent 主界面**：目前设计上以英文为主，不受语言包影响，这是正常现象。
- **经典编辑器窗口**：安装语言包后会显示中文菜单。
- **快速自检**：能看到顶部"文件"就说明语言包已生效。

## 2.5 让 AI 永远用中文回答

### 2.5.1 设置 User Rules

打开设置：`Cursor Settings → Rules → User Rules`

添加以下规则：

```text
Always respond in Simplified Chinese (简体中文). 无论我用什么语言提问，请始终用简体中文回答。
```

### 2.5.2 验证是否生效

新开一个对话，发送：

```
reply with one word
```

如果 AI 回复了中文（如"好"），说明规则已生效。

---

## 2.6 启动设置：直接进入经典模式

### 2.6.1 修改桌面快捷方式

右键桌面 Cursor 图标 → 属性 → 修改"目标"：

```text
E:\软件\cursor\Cursor.exe --classic --open-editor
```

参数说明：
- `--open-editor`：打开经典编辑器窗口
- `--classic`：跳过 Glass，直接进入经典模式

### 2.6.2 如果仍出现 Glass 界面

完全退出 Cursor（打开任务管理器，结束所有 `Cursor.exe` 进程），再用快捷方式重新启动。

## 2.7 AI 核心功能详解

### 2.7.1 Tab 智能补全

这是 Cursor 最常用的功能。当你写代码时，Cursor 会预测你下一步要输入的内容，灰色文字显示建议。

- **接受建议**：按 `Tab`
- **忽略建议**：继续打字或按 `Esc`
- **逐词接受**：按 `Ctrl + →`

**示例：** 你输入 `def calculate_`，Cursor 可能自动补全为 `def calculate_total(items):`，再按 Tab 连方法体都帮你写好。

### 2.7.2 AI Chat（侧边对话）

快捷键：`Ctrl + L`

打开右侧 AI 对话面板，可以：

```
@文件名        引用某个文件内容
@文件夹        引用整个文件夹
选中代码后提问  AI 会聚焦在选中内容上
```

常用对话示例：

```
这段代码是做什么的？
帮我给这个函数写单元测试
这里有没有潜在的性能问题？
帮我把这段代码改成异步版本
```

### 2.7.3 Composer / Agent（直接修改代码）

快捷键：`Ctrl + I`（行内编辑）或 `Ctrl + Shift + I`（全局 Composer）

与 Chat 的区别：
- Chat：AI 给建议，你手动修改
- Composer：AI 直接帮你修改文件，你审核后接受

**使用场景：**

```
帮我重构这个类，把所有方法按功能分组
在这个文件里添加错误处理逻辑
把所有变量名从驼峰命名改成下划线命名
```

### 2.7.4 全代码库问答（Codebase）

在 Chat 里输入 `@Codebase` 可以让 AI 检索整个项目：

```
@Codebase 这个项目是怎么处理用户认证的？
@Codebase 哪个文件负责数据库连接？
@Codebase 帮我找所有用到 fetch 的地方
```

## 2.10 实战案例

### 案例一：用 Cursor 学习一段陌生代码

1. 打开文件，选中看不懂的代码块
2. 按 `Ctrl + L` 打开 Chat
3. 输入"帮我解释这段代码的逻辑"
4. AI 逐行讲解，还可以追问"第三行的意思是什么"

### 案例二：快速修复报错

1. 终端出现红色报错信息
2. 复制报错，在 Chat 里发送"帮我修复这个错误：[粘贴报错]"
3. 或直接选中报错行，右键 → "Fix with AI"

### 案例三：AI 帮你写注释

1. 选中一段没有注释的函数
2. 在 Chat 里输入"帮这段代码加上中文注释"
3. AI 生成注释后点击"Apply"一键应用
