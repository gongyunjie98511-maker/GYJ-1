# 第一章：Claude —— AI 助手入门到精通

> 本章涵盖 Claude 的背景、安装、高效使用技巧与实战案例，适合零基础读者从头学起。

## 1.1 Claude 是什么

### 1.1.1 Anthropic 与 Claude 的诞生

Claude 由美国 AI 公司 **Anthropic** 开发。Anthropic 成立于 2021 年，创始团队来自 OpenAI，核心理念是"负责任地构建 AI"，专注于 AI 安全研究。

Claude 这个名字来源于信息论之父 **Claude Shannon**，象征着严谨与智慧。

### 1.1.2 Claude 的版本演进

| 版本 | 特点 |
|------|------|
| Claude 1 | 早期版本，奠定基础 |
| Claude 2 | 上下文窗口扩大至 10 万 token，支持长文档分析 |
| Claude 3（Haiku / Sonnet / Opus）| 多档位设计，Opus 达到顶级水准 |
| Claude 3.5 Sonnet | 速度与智能的最佳平衡点，广受欢迎 |
| Claude 4（Haiku / Sonnet / Opus）| 目前最新一代，推理与编程能力大幅提升 |

> **Haiku = 快速轻量，Sonnet = 均衡推荐，Opus = 最强但较慢**

### 1.1.3 Claude 能做什么

| 能力 | 具体场景 |
|------|----------|
| 写作辅助 | 写文章、改稿、润色、翻译 |
| 代码编程 | 写代码、调试、解释代码、重构 |
| 数据分析 | 分析表格、总结报告、提炼要点 |
| 学习助手 | 解释概念、出题练习、答疑解惑 |
| 创意创作 | 写故事、写剧本、头脑风暴 |
| 文件处理 | 读取 PDF、总结长文、提取信息 |

### 1.1.4 Claude vs 其他 AI

| 对比项 | Claude | ChatGPT | Gemini |
|--------|--------|---------|--------|
| 开发公司 | Anthropic | OpenAI | Google |
| 上下文长度 | 极长（200K token）| 较长 | 较长 |
| 代码能力 | 强 | 强 | 较强 |
| 中文支持 | 良好 | 良好 | 良好 |
| 安全性重视 | 极高 | 高 | 高 |
| 免费额度 | 有限制 | 有限制 | 有限制 |

## 1.2 注册账号

### 1.2.1 注册步骤

1. 打开浏览器，访问 `https://claude.ai`
2. 点击 **Sign Up**
3. 选择注册方式：
   - 用 **Google 账号** 一键登录（推荐，最方便）
   - 用 **邮箱** 注册，填写邮箱后验证
4. 完成后进入对话界面即注册成功

### 1.2.2 免费版与付费版区别

| 功能 | 免费版 | Pro 付费版（约 $20/月）|
|------|--------|----------------------|
| 使用模型 | Sonnet（限额）| 全系列模型 |
| 每日消息数 | 有上限 | 大幅提升 |
| 优先访问 | 否 | 是（高峰期不排队）|
| Projects 功能 | 基础 | 完整 |
| 文件上传 | 有限 | 更多 |

> 初学者免费版完全够用，熟悉后再考虑升级。

## 1.3 安装 Claude 桌面应用

### 1.3.1 方式一：官方安装包（推荐）

1. 登录 `claude.ai` 后，页面会提示下载桌面版
2. 下载 `Claude Setup.exe`（约 7MB）
3. 双击运行，自动安装完成
4. 桌面出现 Claude 图标，双击即可启动

### 1.3.2 方式二：Chrome PWA 安装

PWA（Progressive Web App）是把网页"安装"成桌面应用，体验接近原生软件。

1. 用 Chrome 浏览器打开 `https://claude.ai` 并登录
2. 点击地址栏右侧的 **安装图标**（电脑形状）
3. 弹窗点击"安装"
4. 桌面出现 Claude 快捷方式，打开后没有地址栏

### 1.3.3 两种方式的区别

| 对比 | 官方安装包 | PWA |
|------|-----------|-----|
| 安装大小 | 较小 | 依赖 Chrome |
| 更新方式 | 自动更新 | 跟随网页自动更新 |
| 独立性 | 独立运行 | 依赖 Chrome 内核 |
| 推荐程度 | ★★★★★ | ★★★★☆ |

## 1.4 Claude 对话技巧

掌握提问技巧，可以让 Claude 的回答质量提升数倍。

### 1.4.1 基本原则：清晰 + 具体

**差的提问：**
> 帮我写个介绍

**好的提问：**
> 帮我写一段 200 字的自我介绍，用于求职，我是应届生，学计算机专业，性格内向但认真负责，目标岗位是后端开发。

**原则：** 说清楚**是什么、给谁看、什么风格、多少字、有什么限制**。

### 1.4.2 给 Claude 一个"角色"

```text
你是一位有 10 年经验的 Python 工程师，请用简洁的语言帮我解释以下代码……
```

```text
你是一位小学语文老师，请用小学生能理解的方式解释"量子力学"是什么。
```

角色设定能让 Claude 的回答更聚焦、更专业。

### 1.4.3 分步骤提问

遇到复杂任务，不要一次性全部提问，而是**分步引导**：

```
第一步：先帮我列出写商业计划书的框架
第二步：（收到框架后）帮我展开"市场分析"这一节
第三步：（继续）帮我把语气调整得更正式
```

### 1.4.4 要求 Claude 用特定格式输出

```text
请用 Markdown 表格的形式输出
请用编号列表输出，每条不超过 20 字
请输出 JSON 格式，包含 name、age、city 三个字段
```

### 1.4.5 让 Claude 检查自己的回答

```text
请再检查一遍，确认没有逻辑错误
上面的代码有没有潜在的 bug？
```

### 1.4.6 常用场景提示词模板

**翻译：**
```text
请将以下英文翻译成简体中文，保持原文语气，不要意译：
[粘贴原文]
```

**代码调试：**
```text
以下代码运行后报错，错误信息是：[粘贴错误]
请帮我找出原因并给出修复方案：
[粘贴代码]
```

**总结长文：**
```text
请用 5 条要点总结以下文章，每条不超过 30 字：
[粘贴文章]
```

## 1.5 安装 Claude Code（命令行版）

### 1.5.1 Claude Code 是什么

Claude Code 是运行在终端（命令行）里的 Claude，专为程序员设计。它可以：

- 直接读取和修改你电脑上的代码文件
- 在项目目录里理解整个代码结构
- 帮你执行命令、查看输出、持续迭代
- 不需要来回复制粘贴代码

### 1.5.2 安装前提：Node.js

Claude Code 基于 Node.js 运行，需先安装：

```powershell
# 方式一：用 winget 安装（推荐）
winget install OpenJS.NodeJS

# 验证安装成功
node --version    # 应显示 v18 或更高
npm --version
```

### 1.5.3 安装 Claude Code

```powershell
npm install -g @anthropic-ai/claude-code
```

安装完成后验证：

```powershell
claude --version
```

### 1.5.4 首次登录

```powershell
claude
```

首次运行会弹出登录提示，用浏览器打开链接，用 claude.ai 账号授权即可。

### 1.5.5 基本使用方式

```powershell
# 进入你的项目文件夹，再启动
cd E:\practice
claude
```

启动后直接输入问题：

```
> 帮我看一下 README.md 的结构
> 帮我在 Git 文件夹里新建一个 Git进阶.md
> 帮我把所有 .md 文件列出来
```

### 1.5.6 Claude Code 常用斜杠命令

| 命令 | 作用 |
|------|------|
| `/help` | 查看所有可用命令 |
| `/clear` | 清空当前对话上下文 |
| `/exit` | 退出 Claude Code |
| `/review` | 审查当前代码改动 |
| `/init` | 为项目生成 CLAUDE.md 说明文件 |

### 1.5.7 Claude Code vs 网页版 Claude

| 对比 | Claude Code（终端）| Claude 网页版 |
|------|-------------------|--------------|
| 文件操作 | 直接读写本地文件 | 需手动上传下载 |
| 适用场景 | 编程、项目开发 | 写作、问答、分析 |
| 上手难度 | 需要会用终端 | 零门槛 |
| 效率 | 编程场景极高 | 通用场景高 |

## 1.6 实战案例

### 案例一：用 Claude 写周报

```text
我本周的工作：
1. 完成了用户登录模块的开发
2. 修复了 3 个线上 bug
3. 参加了项目评审会议

请帮我写一份 200 字以内的工作周报，语气正式，突出成果。
```

### 案例二：用 Claude 学新技术

```text
我是一个会 Python 的程序员，想学 Docker。
请先给我解释 Docker 是什么（用类比方式），
然后列出学习路线，每步不超过一行。
```

### 案例三：用 Claude Code 整理笔记仓库

在终端启动 Claude Code 后：

```
> 帮我查看 E:\practice 下有哪些文件夹
> 帮我在 README.md 里添加一个新章节的目录项
> 帮我把修改提交到 Git 并推送到 GitHub
```

## 1.7 注意事项

1. **不要输入真实密码、银行卡号等敏感信息**，Claude 虽然安全，但养成习惯很重要。
2. **AI 可能出错**：对于重要信息（医疗、法律、财务），务必核实后再使用。
3. **版权意识**：Claude 生成的内容用于商业用途前，了解所在平台的使用条款。
4. **保持对话连贯**：在同一个对话窗口内继续提问，Claude 能记住上下文；新开对话则从零开始。
